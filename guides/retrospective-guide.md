# 历史分析复盘指南

> **免责声明**：复盘结论与经验库仅供研究参考，**不构成投资建议**。

## 目的

用**当日最新资讯**对照**历史报告中的可验证判断**，标注对错、分析偏差原因，并将经验写入 `reports/_lessons/lessons-learned.md`，持续改进后续分析质量。

## 读取范围

按时间倒序查找，默认回看 **7 个自然日**（用户可指定 14/30 日）：

```
reports/<YYYY-MM-DD>/daily-research-report.md     # 主日报
reports/<YYYY-MM-DD>/<sector>/**/report.md      # 个股报告（若存在）
reports/_lessons/lessons-learned.md               # 累积经验 + 待验证清单（跨日）
```

若目录下无任何历史日报 → 进入**首跑模式**（见下）。

## 提取可验证判断

从历史报告中抽取具备**事后可核对**的陈述，分类为：

| 类型 | 示例 | 验证方式 |
|------|------|----------|
| 方向判断 | 「中海油中性偏多」 | 股价涨跌、研报评级变化、业绩公告 |
| 宏观变量 | 「布伦特 ~$77」 | 当日油价检索 |
| 景气预期 | 「光伏产能出清未完成」 | 龙头季报、行业价格、政策 |
| 估值结论 | 「中石化 PB 低估」 | PE/PB 变化、分红实施 |
| 风险触发 | 「油价跌破 $70 将下修盈利」 | 是否已发生 |
| 相对排序 | 「海油>中石油>中石化」 | 区间涨跌幅对比 |

**不验证**纯描述性事实（如「Q1 营收 547 亿」）除非用于检验当时引用的数据是否准确。

## 验证结论标签

| 标签 | 含义 |
|------|------|
| ✅ 验证正确 | 最新资讯与此前判断一致 |
| ⚠️ 部分正确 | 大方向对但幅度/时点偏差 |
| ❌ 验证错误 | 判断与事实相反或明显失准 |
| ⏳ 待验证 | 判断周期未到，继续跟踪 |

每条须附：**原判断摘要 · 验证依据（来源+日期）· 偏差原因（若适用）**

## 偏差原因归类

复盘时从下列选 1–2 项标注根因，便于经验沉淀：

- 宏观变量突变（地缘、政策、利率）
- 数据滞后（用了过期财报/油价）
- 情绪溢价/折价误判
- 传导链条简化过度
- 检索样本不足或来源单一
- 时间窗口过短/过长
- 板块内部分化未识别

## 经验提炼规则

从复盘产出 **1–5 条**可执行经验，写入 `lessons-learned.md`：

**好的经验**（具体、可复用）：
> 石油板块：油价单周波动 >5% 时，不立即调整三桶油排序，等 IEA/EIA 月报确认后再修正。

**差的经验**（过于笼统，禁止）：
> 要加强研究。

每条格式：
```markdown
- **{YYYY-MM-DD}** | {板块} | {验证结果} — {原判断一句话} → {经验一句话}
```

同时更新「待验证判断」表：
- 已验证 → 删除或标为已结案
- 今日新判断 → 追加一行，默认验证截止 = 报告日 +30 日（宏观变量 +7 日）

## 首跑模式

无历史日报时，复盘章节写：

1. 说明本次为基线建立，无历史可对照
2. 从当日报告中提取 **5–10 条**核心可验证判断，录入「待验证清单」
3. 不编造虚假复盘

## 检索补充（验证专用）

对「待验证」中已到期的条目，额外检索：

```
{name} {code} 股价 {start_date} to {end_date}
{sector} {macro_variable} actual vs forecast {month}
{原判断关键词} 最新 结果
```

与当日宏观/个股资讯合并，形成验证依据。
